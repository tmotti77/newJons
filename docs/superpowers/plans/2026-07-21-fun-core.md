# Fast Lane — Fun Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the planning and reveal screens into a flat-vector, tap-to-travel experience so a Quick game feels like moving through a living town with real time-pressure and a reveal that roasts players by name.

**Architecture:** Client + content only. The pure engine already models the town (`LocationId`, `travel` action, `PlayerState.location`, travel costs, per-week ledger), so the new map just emits the same `Action[]` the server already validates, and the reveal renders the same `round_results` the server already writes. Testable logic (the reveal-card picker) lives in `packages/engine` where vitest exists; RN screens are verified by typecheck + `expo export` + on-device play.

**Tech Stack:** Expo SDK 54, React 19, React Native 0.81, expo-router 6, TypeScript strict, zustand, i18next (en+he), `react-native-svg` (added in Task 1), vitest (engine only).

## Global Constraints

- Engine stays PURE: no I/O, no `Date.now()`, no `Math.random()` (use `rng.ts`), no imports from `apps/` or `supabase/`. (`CLAUDE.md`)
- All user-facing strings are i18n keys in `en.json` + `he.json`. Never hardcode copy. Verify RTL. (`CLAUDE.md`)
- All tunable game numbers live in `packages/engine/src/config/balance.ts` only. (`CLAUDE.md`)
- Server is authoritative; clients never write game state directly. This milestone changes NO server code and NO DB. (spec §5)
- Every new engine behavior ships with vitest tests; the determinism suite must stay green. (`CLAUDE.md`)
- The 11 `LocationId`s are exactly: `home`, `burgerBarn`, `college`, `gadgetCity`, `flipIt`, `dressCode`, `careerHub`, `bank`, `quickMart`, `theSpot`, `rentALord`. (`packages/engine/src/types.ts`)
- Location icons must be distinct, polished flat-vector marks — not the placeholder emoji/glyphs. Named acceptance item. (spec §4.1, §9.5)
- Run from repo root before declaring any task done: `pnpm test` (engine+shared green), and from `apps/mobile`: `npx tsc --noEmit`. (`CLAUDE.md`)

**Branch:** `feat/fun-core` (already created, built on the SDK 54 upgrade). Commit after every task.

      <Circle cx={17} cy={17} r={1.4} fill={S} />

  // …bank (columns+$), gadgetCity (screen), theSpot (disco), quickMart (cart),
